How to Test the AI Receptionist

Step 1 — Register your clinic in Tab 1. Copy the tenant_id.

Step 2 — Add doctors in Tab 2 using that tenant_id.

Step 3 — Point your Twilio phone number's Voice webhook to:
https://YOUR_NGROK_URL/v1/webhooks/call/answer?call_id=NEW

Step 4 — Call the Twilio number. The AI receptionist will greet you, tell you about available doctors, and book appointments — all stored in MongoDB.

What the AI can do on the call:

  • Greet callers professionally as a named receptionist
  • Tell callers which doctors are available and their specialties
  • Collect patient name, preferred doctor, reason, and date/time
  • Confirm all details before booking
  • Save appointment with patient details + doctor_id to MongoDB
  • Read back the Appointment ID so the patient can reference it
  • Handle inactivity gracefully with a keepalive message
  • Say a warm goodbye when the call ends
API Endpoints:
POST /v1/tenants/register — Register clinic
POST /v1/doctors/register — Add doctor
GET /v1/doctors?tenant_id=... — List doctors
GET /v1/appointments — View all appointments
GET /docs — Swagger UI (dev mode)
